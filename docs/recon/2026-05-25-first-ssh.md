## Headline findings

- **`eips` lives at `/usr/sbin/eips`.** Non-interactive ssh sessions get
  `PATH=/usr/bin:/bin`, so `which eips` reports nothing. Any script we
  install must either set `PATH=/usr/sbin:$PATH` or call `eips` by full
  path. Build size: 57 KB. Built 2021-12-03 (same date as firmware).
- **`crond` is already running (PID 893).** Root's crontab is a single
  file at `/etc/crontab/root` (busybox-cron style, classic `m h dom mon
  dow command` syntax). The system already uses it for things like
  `/usr/sbin/tinyrot` every 15 min. Installing our refresh job = append
  one line to this file (after `mntroot rw`).
- **Display: 600 Ă— 800, 8 bpp grayscale framebuffer.** Confirmed by
  `eips -i`: `xres=600 yres=800 bits_per_pixel=8 grayscale=1`. The panel
  itself dithers to 16 levels, but the framebuffer accepts 8-bit gray.
  We should render **8-bit grayscale PNGs** (not 4-bit) â€” same data, no
  on-device conversion. `line_length=608` (16-aligned stride).
- **SSH server: Dropbear v2020.81.** Multiple dropbear processes alive
  (one accepting, one or two for current sessions). Legacy `ssh-rsa`
  required (matches existing ssh config).
- **Rootfs:** kernel cmdline says `ro`, but `mount` shows `rw` right
  now â€” the jailbreak (winterbreak2) appears to leave it remounted
  read-write after boot. Don't rely on this; **always `mntroot rw`
  explicitly before editing system files**, and `mntroot ro` after.
- **`fbink` is installed via the `linkss` extension** at
  `/mnt/us/linkss/bin/fbink`. NiLuJe's excellent framebuffer renderer
  with fonts, layout, etc. We don't need it for v1 (server renders the
  whole image) but it's a great escape hatch for on-device debug/status
  overlays.

## Hardware / framework facts (for future reference)

- Kernel: `Linux 3.0.35-lab126 #2 PREEMPT armv7l`. ARMv7, 32-bit. Go cross-compile target = `GOOS=linux GOARCH=arm GOARM=7` if we ever want to ship a binary to the device (we don't, for v1).
- Firmware: `Kindle 5.12.2.2` (build `juno_120202_bourbon_wario-379151`, 2021-12-03).
- Memory: 255 MB total, ~93 MB free at idle. Tight; don't run anything heavy on the device.
- Framebuffer device: `/dev/fb0`. Driver: `mxc_epdc_fb`. Smem 3.27 MB.
- Display kernel param: `video=mxcepdcfb:E60,bpp=8,x_mem=3M` â€” `E60` is the 6" Pearl panel.
- Networking: `wlan0` = `10.0.0.178` (DHCP from `10.0.0.1`). `usb0` = `192.168.15.244` (usbnet extension, used when the Kindle is plugged into a host).
- Running framework procs: `lipc-daemon`, `powerd`, `awesome` (window manager), `kb`. The reader UI sits on top of these.
